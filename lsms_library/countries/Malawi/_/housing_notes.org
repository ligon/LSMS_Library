#+TITLE: Housing Feature Implementation Notes
#+DATE: 2026-04-12

* Design decisions

- Schema matches Uganda: =(t, i)= index, =Roof: str=, =Floor: str=.
- Categorical string values (material names), not binary indicators.
  Consumers wanting binary derive trivially: =df['Roof'] == 'Grass'=.
- Labels normalized to Title Case for cross-wave consistency.

* Notes for extending to other countries

** Module letter is NOT stable across surveys
   - Malawi: Module G (IHS2 2004-05), Module F (IHS3+ 2010--)
   - Uganda: GSEC9 / GSEC11 (varies by wave)
   - ALWAYS check the WB data dictionary: replace =get-microdata= with
     =data-dictionary= in the catalog URL from =Documentation/SOURCE.org=.

** Common Stata variables
   Housing materials are typically 3 questions (walls, roof, floor) in one
   module. Variable names follow the module prefix.

** Case normalization
   Label casing varies across waves even within the same country.
   Use inline =mapping:= dicts in =data_info.yml= to normalize to Title Case.
   Could consolidate into =categorical_mapping.org= tables named =Roof= and
   =Floor= if the mapping grows unwieldy.

** Cross_Sectional + Panel merge (Malawi 2016-17, 2019-20)
   IHS4/IHS5 have separate CS and Panel files. Follow the =household_roster=
   pattern: list both files with Panel overriding =i:= to the panel ID
   (=y3_hhid=, =y4_hhid=).

** Walls column
   Data is available (=hh_f07= / =g08= in Malawi) but excluded from the
   current schema to match Uganda. If added later, update =data_scheme.yml=
   and add =Walls:= to each wave's =data_info.yml=.
