from lsms.tools import from_dta
import dvc.api
import pyreadstat
import pandas as pd
from cfe.df_utils import df_to_orgtbl

Country = 'Ethiopia'

fs = dvc.api.DVCFileSystem('./')

itemlabel_location = {'2011-12':('sect5a_hh_w1.dta','hh_s5aq00'),
                      '2013-14':('sect5a_hh_w2.dta','hh_s5aq00'),
                      '2015-16':('sect5a_hh_w3.dta','hh_s5aq0a'),
                      '2018-19':('sect6a_hh_w4.dta','item_cd')}

itemlabels = {}
for k,v in itemlabel_location.items():
    try:
        with fs.open(f'/{Country}/{k}/Data/{v[0]}') as dta:
            df = from_dta(dta)
            itemlabels[k] = df[v[1]].value_counts().index.tolist()
    except ValueError:  # pd.io.stata can't read some versions of dta files...
        fs.get_file(f'/{Country}/{k}/Data/{v[0]}',f'/tmp/{v[0]}')
        df, meta = pyreadstat.read_dta(f'/tmp/{v[0]}')
        itemlabels[k] = list(meta.value_labels[v[1]].values())

# Build dictionary of dictionaries for matching
from collections import defaultdict
import difflib
from ethiopia import Waves

D = defaultdict(dict)
D.update({k:{'Preferred Label':k} for k in itemlabels['2018-19']})

for wave in Waves.keys():
    for l in itemlabels[wave]:
        k = difflib.get_close_matches(l,itemlabels['2018-19'])
        if len(k):
            D[k[0]][wave] = l
        else:
            D[l][wave] = l

df = pd.DataFrame(D).T

print(df_to_orgtbl(df))

- What is missing :: A Food Composition Table (FCT) mapping food items to
  nutritional content (calories, protein, fat, etc.) has not been assembled
  for Ethiopia. The =fct_tools.py= module exists and can pull nutrients from
  USDA FoodData Central, but a complete FCT keyed to Ethiopia's food item
  labels has not been built.
- What exists :: =conversion_to_kgs.json= is complete, built from
  wave-specific =Food_CF_Wave{1..5}.dta= files (present for all five waves
  2011-12 through 2021-22). This handles unit-to-kilogram conversion.
  =fct_tools.py= provides helper functions for pulling nutrient data from
  USDA FoodData Central. =nutrition.py= and a =!make= entry in
  =data_scheme.yml= indicate the pipeline skeleton exists.
- Survey documentation :: Household questionnaires (DVC-tracked PDFs in
  each wave's =Documentation/=) include food consumption modules but do
  not embed conversion factors for nutritional content. The
  =Food_CF_Wave*.dta= files only provide weight conversion factors
  (units to kg), not nutritional composition.
- What is needed to fix ::
  1. Assemble an FCT mapping each "Preferred Label" from =food_items.org=
     to nutrient values (kcal, protein, fat, carbohydrate per 100g).
  2. The FAO resource above provides East African food composition data
     that could serve as a primary source.
  3. Alternatively, use =fct_tools.py= with USDA FoodData Central API keys
     to pull nutrient data, matching FDC IDs to each food item.
  4. The resulting table should be stored as a structured file (CSV or JSON)
     in =Ethiopia/_/= and integrated with =nutrition.py=.

* Sampling Design
** Overview

Two-stage stratified cluster sample. Enumeration Areas (EAs) are
the primary sampling unit (PSU). The Ethiopia Socioeconomic Survey
(ESS) is part of the LSMS-ISA program, covering rural and small-town
areas in waves 1-3, expanding to include all urban areas from wave 2
onward.

ESS data are collected using a post-planting / post-harvest (PP/PH)
design, but the =sample= feature uses the household-level cover page
(=sect_cover_hh_w{N}.dta=), which provides one row per household per
wave (not per visit).

World Bank Microdata Library catalog entries:
- Wave 1 (2011-12): https://microdata.worldbank.org/index.php/catalog/2053
- Wave 2 (2013-14): https://microdata.worldbank.org/index.php/catalog/2247
- Wave 3 (2015-16): https://microdata.worldbank.org/index.php/catalog/2783
- Wave 4 (2018-19): https://microdata.worldbank.org/index.php/catalog/3823
- Wave 5 (2021-22): https://microdata.worldbank.org/index.php/catalog/6161

** Strata

Ethiopia has no explicit strata variable. Strata are constructed as
Region x Urban/Rural (=saq01= x =rural= or =saq14=), following the
ESS sampling documentation. The number of strata varies by wave
because the coverage expanded over time:

| Wave    | Region variable | Rural variable | Strata count | Notes                    |
|---------+-----------------+----------------+--------------+--------------------------|
| 2011-12 | =saq01=         | =rural=        |           18 | No Addis Ababa, Dire Dawa Urban |
| 2013-14 | =saq01=         | =rural=        |           21 | Adds Addis Ababa, Dire Dawa Urban |
| 2015-16 | =saq01=         | =rural=        |           21 | Same coverage as wave 2  |
| 2018-19 | =saq01=         | =saq14=        |           21 | ALL CAPS in source data  |
| 2021-22 | =saq01=         | =saq14=        |           19 | Tigray excluded (conflict) |

The =rural= column in waves 1-3 has varying labels: "rural" / "small
town" (w1), "Rural" / "Small town (urban)" / "Large town (urban)" (w2),
"RURAL" / "SMALL TOWN" / "Meduim and large town" (w3). All non-rural
categories are mapped to "Urban".

Strata labels are harmonized across waves via =categorical_mapping.org=
(fixes casing and spelling differences in region names).

** Cluster (PSU) identifier

The cluster identifier (=v= in the library's index convention) is the
Enumeration Area ID:

| Wave    | Variable  | Type   |
|---------+-----------+--------|
| 2011-12 | =ea_id=   | string |
| 2013-14 | =ea_id2=  | string |
| 2015-16 | =ea_id2=  | string |
| 2018-19 | =ea_id=   | string |
| 2021-22 | =ea_id=   | string |

Waves 2-3 use =ea_id2= (the wave-2 re-identified EA), while waves
1, 4, and 5 use =ea_id=. The EA ID format is a long numeric string
(e.g., "010101088801601") encoding region/zone/woreda/town/EA.

** Sampling Weights
*** Weight variables per wave

All weights are in the household cover page (=sect_cover_hh_w{N}.dta=).
Each wave provides a single weight variable; there is no separate
panel weight.

| Wave    | Weight variable | Coverage          | Min   | Max      |
|---------+-----------------+-------------------+-------+----------|
| 2011-12 | =pw=            | 3969/3969 (100%)  |  7.76 | 17094.66 |
| 2013-14 | =pw2=           | 5262/5262 (100%)  |  6.90 | 22097.78 |
| 2015-16 | =pw_w3=         | 4954/4954 (100%)  |  6.84 | 32753.24 |
| 2018-19 | =pw_w4=         | 6770/6770 (100%)  |  7.59 | 25432.47 |
| 2021-22 | =pw_w5=         | 4959/4959 (100%)  |  4.39 | 32370.29 |

Since only one weight is provided per wave, both =weight= and
=panel_weight= in the =sample= table contain the same values. The
BID documents (available under "Related Materials" in each catalog
entry) describe the weight construction methodology.
