* Phase 3 --- Remove =v= from feature configs (do first)

** Goal

Features that currently bake =v= into their index stop doing so.
Parquets will be built with only their natural index (=(i, t, ...)=
without =v=).

** Scope: data_scheme.yml changes

For every country that has =sample= in its =data_scheme.yml=,
remove =v= from the =index:= declaration of all features *except*
=cluster_features= and =sample= (which legitimately own =v=).

Example --- Uganda before:

    index: (t, v, i, pid)
#+end_example

After:
#+begin_example
  household_roster:
    index: (t, i, pid)
#+end_example

** Scope: data_info.yml changes

Remove =dfs:= merge blocks that exist solely to join =v= from cover
pages.  The pattern (from CLAUDE.md) is:

#+begin_example
household_roster:
    dfs:
        - df_roster
        - df_cluster        # <-- delete this sub-df
    df_roster: ...
    df_cluster:
        file: ../Data/HH/gsec1.dta
        idxvars:
            i: hhid
        myvars:
            v: s1aq04a      # <-- only purpose is to provide v

        - i

        - t
        - v                 # <-- remove v
        - i
        - pid

After: the =dfs:= structure collapses to a simple single-file entry,
or the =df_cluster= sub-df is deleted and =final_index= drops =v=.

** Scope: legacy .py scripts

Scripts like Uganda's =earnings.py=, =enterprise_income.py=, and
=food_prices_quantities_and_expenditures.py= that manually join =v=
from =household_roster()= or =cluster_features()= --- remove those

** What NOT to change

- =cluster_features= --- legitimately indexed by =(t, v)=
- =sample= --- legitimately has =v= as a column
- Countries without =sample= in =data_scheme.yml= --- leave alone
- =!make= features that only declare the tag (no explicit index) ---
  these need their scripts inspected individually

* Phase 2 --- Framework joins =v= from =sample()= (do second)

** Goal

When a feature's =data_scheme= declares =v= in its index, and the
built DataFrame doesn't have =v=, the framework transparently joins
it from =sample()= at API time.

** Implementation: =_join_v_from_sample()=

New method on =Country=:

#+begin_src python
def _join_v_from_sample(self, df: pd.DataFrame) -> pd.DataFrame:
    """Join cluster identity (v) from sample() onto df.

    Only fires when sample is in data_scheme and df has (i, t) but
    not v in its index.
    """
    cache = getattr(self, '_sample_v_cache', None)
    if cache is None:
        try:
            s = self.sample()
            cache = s[['v']].copy()
            self._sample_v_cache = cache
        except Exception:
            self._sample_v_cache = False
            return df
    if cache is False:
        return df

    idx_names = list(df.index.names)
    flat = df.reset_index()
    # Coerce join keys to string for safe matching
    for key in ['i', 't']:
        if key in flat.columns:
            flat[key] = flat[key].astype(str)
    v_lookup = cache.reset_index()
    v_lookup['i'] = v_lookup['i'].astype(str)
    v_lookup['t'] = v_lookup['t'].astype(str)

    flat = flat.merge(v_lookup, on=['i', 't'], how='left')

    # Insert v after t in the index
    new_idx = []
    for n in idx_names:
        new_idx.append(n)
        if n == 't':
            new_idx.append('v')
    if 'v' not in new_idx:
        new_idx.append('v')
    return flat.set_index(new_idx)
#+end_src
