* Pivot — wave-level L2 cache for YAML-path tables

Diagnostic work on the 1315-row =log HSize= drift
(=household_characteristics=) was pre-empted by a more systemic
finding: the 43-min plateau under =market='Region'= across 12
pytest-xdist workers is caused by =_market_lookup= iterating all 8
Uganda waves calling wave-level =cluster_features()=.  Wave-level
=cluster_features= is a YAML-path table and, per design, has no L2
parquet cache — so every call runs the YAML grabber + reads the
cover-page =.dta= via L1 (DVC SQLite).  12 workers × 8 waves =
concurrent DVC-SQLite contention.

User's design decision: extend L2 caching to wave-level YAML-path
tables (pre-transform; same staleness semantics as country L2;
=to_parquet= gets an =absolute_path= flag to suppress the call-
stack heuristic).  Staleness detection is a v0.8.0 concern.
