  gives 712 unique district-community combinations. =region= (string) is complete.
- 2009-10: =ppweight3= appears to be a per-person weight (mean ~3.8x larger than =hhweight3=),
  not a panel weight. Wave 1 is a fresh cross-section; no panel weight applies.

* 7. Open Questions (require human decision before execution)

1. *Weight variables for 2013-14 and 2017-18*: No sampling weight variable was found in any
   on-disk =.dta= file for these two waves. The BID PDFs (accessible via DVC) may document where
   they are. An agent should pull and read them before attempting the =sample= feature. Alternatively,
   the wave may not have been designed with published expansion weights (common in academic panels).

2. *2013-14 EA coding*: The FPrimary 9-digit structure encodes EA implicitly. Is it safe to use
   =FPrimary[:6]= as the cluster identifier, or is there a canonical EA code in a file we haven't
   found? (The =01ea_employmentscreener.dta= file has =eacode= and =regioncode= columns but they
   are all empty strings.)

3. *Panel structure across waves*: =ghanasps.py= documents that 2013-14 uses =FPrimary= and 2017-18
   tracks returning households via =FPrimary_original=. What fraction of 2009-10 households were
   re-interviewed in 2013-14 and 2017-18? This affects whether =panel_weight= is meaningful vs.
   identical to =weight=.

4. *Catalog IDs for 2013-14 and 2017-18*: The 2009-10 catalog ID is 2534. The 2013-14 DDI file
   references the same ID, but the 2017-18 wave has no documentation or LICENSE.org at all. A human
   must verify the correct WB catalog entry for each wave before adding SOURCE.org files.

5. *Library registration*: There is no =data_scheme.yml= and therefore no library registration.
   When GhanaSPS is added, it should be confirmed whether it should appear in the =Feature=
   cross-country API alongside Uganda, Nigeria, Tanzania, etc. (it currently doesn't).

6. *Wave-1 =ppweight3= interpretation*: Is =ppweight3= a per-person expansion weight or a panel
   weight? The ratio to =hhweight3= equals approximately household size (mean ratio = 3.8). If
   it is indeed per-person, both =weight= and =panel_weight= should map to =hhweight3=.

7. *food_quantities=*: The CONTENTS.org notes a =conversion_to_kgs.json= as TODO. The West Africa
   Food Conversion Table is referenced by URL (Google Sheets). An agent would need human help to
   materialize this table before =food_quantities= can be built.
