* Key learning (why plain `pytest` skipped the invariance tests)
- =conftest.py:96-109= : in DEFAULT mode pytest PURGES the Uganda cache at
  session start (=_purge_country_caches("Uganda")=). So a plain
  =pytest test_uganda_invariance.py= wipes the parquets you just built ->
  "No Uganda parquets built" -> 53 skips. Use =--no-purge= to validate a
  hand-built cache. The same fragility (invariance tests only READ parquets,
  depend on another test rebuilding them first) is the #330 flakiness and why
