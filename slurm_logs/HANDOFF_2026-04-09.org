** Design note: =sample()= as single source of truth for =v=

=slurm_logs/DESIGN_sample_as_v_source.org= documents the 4-phase
migration plan:
1. Build =sample= for all countries (Phase 1 --- this session)
2. Framework joins =v= from =sample= at API time (Phase 2 --- next)
3. Remove =v= from feature parquets and YAML configs (Phase 3)
4. Simplify =_add_market_index= (Phase 4)

This is the key architectural change: parquets store =(i, t, ...)=
without =v=, and the framework joins =v= transparently from =sample()=.
=market='Region'= then chains =(t, v) -> Region= from =cluster_features=.

* Lessons learned

** BID first
Always read the Basic Information Document before inspecting data.
Albania turned out to have a panel component (2003-04) despite
appearing to be independent cross-sections.  GhanaLSS had
incompatible weight types across rounds (probability vs expansion).

** One agent per country
Never batch multiple countries into one agent.  Each country requires
independent data inspection, BID reading, and testing.  Batching
means sequential processing on one core while others sit idle.

** Scatter-gather orchestration
The manual pattern this session (dispatch N agents, commit as results
arrive) should be codified.  A coordinator agent could: discover
countries needing work, dispatch one agent per country in a single
message, process notifications, commit/redispatch.

** Cross-sectional vs panel weights
Surveys with rotating panels provide two distinct weights.  The BID
documents the construction.  Look for =rotate= variable to identify
refreshment households.  The cross-sectional weight covers everyone;
the panel weight is NaN/zero for new entrants.

** =format_id= for numeric cluster/strata IDs
Numeric IDs passing through =myvars= (not =idxvars=) retain float
type and get =.0= suffixes.  Use =format_id= or the framework's
=_normalize_v()= to strip these before joins.
