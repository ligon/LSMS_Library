* Workflow that worked: worktree delegation + coordinator build-verify
- Agents run in isolated git worktrees, *static-verify only* (read code,
  grep, parse YAML, read .dta labels via pyreadstat/get_dataframe), commit
  on =fix/NNN=, push.  They do NOT build.
- Why: the venv =.pth= hardcodes the MAIN checkout path, so any
  =import lsms_library= in a worktree runs main's code + reads main's
  =countries/=.  Worktree agents literally cannot build-verify their edits.
- The coordinator build-verifies each pushed branch IN THE MAIN CHECKOUT via
  =git checkout -B vNNN origin/fix/NNN= (a *differently-named* local branch
  from the origin ref — the agent's worktree still holds =fix/NNN=, so a
  same-name checkout fails), then PR -> merge -> *close the issue* (merging to
  development does NOT auto-close).
- The gate earns its keep: static-only agents shipped build bugs the
  coordinator caught — e.g. =convert_categoricals= defaults (India/KZ assets,
  1999 roster), an undeclared =food_acquired= (Cambodia/Serbia), a
  DataFrame-truthiness crash (Mali #380).
- Branches touching the same files (e.g. two EthiopiaRHS config branches)
  conflict at merge: merge one, then =git merge origin/development= into the
  other and resolve (both blocks coexist).
