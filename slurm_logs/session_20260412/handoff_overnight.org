** A. Framework follow-ups (Phase 1 remainder)

*** Kroeber decomposition missing from =Feature('household_roster')=

Per the framework diagnosis: stale caches written before kinship
expansion landed don't have Generation/Distance/Affinity; the cache
best-effort read bypasses re-derivation from =Relationship=.  Fix
options: (a) invalidate all roster caches and rebuild; (b) make
=_expand_kinship= idempotent AND run on cached reads; (c) add cache
versioning in v0.8.0.

*** Wave-level transforms not re-applied on cached reads

The Benin =shocks(df)= function explicitly =df.drop(columns=cope_cols)=
and maps =Affected*= to bool, but these aren't re-applied on cached
reads.  The audit found 26 Cope* columns leaking and 4.6M rows with
null Affected*.  Fix: either cache-invalidate all affected shocks
parquets + rebuild, or add wave-transform re-apply in
=_finalize_result()= (requires idempotency audit of every country's
=def shocks(df)= function — risk of double-transform KeyError on
re-run).

*** Sex spelling normalization not reaching derived features

=household_characteristics= produces 69 fragmented Sex columns (M, F,
Male, Masculino, Feminin, ...) because the derivation pivots on Sex
values BEFORE =_enforce_canonical_spellings()= folds them.  Fix:
either normalize before pivoting in =transformations.py=, or expand
the spellings set in =data_info.yml=.

*** =int_t= lowercase typo across 10/11 countries

Canonical is =Int_t= (datetime).  Only Mali's =interview_date=
produces the canonical form.  Everywhere else it's =int_t= (string,
not datetime64).  Also missing =v= in the index level for most
countries.  Fix scope: 10 wave-level =data_info.yml= files to touch,
maybe a framework rename helper.

** B. Design questions (per user's explicit deferral)

*** =food_acquired= canonical-columns structure

54 columns vs canonical 3 (=Quantity=, =Price=, =Produced=).
Countries ship market / farmgate / unitvalue_home / unitvalue_away /
own / inkind variants.  User wants to discuss — reflects real
cross-country survey-instrument variation.

*** Uganda / Tanzania / (other) =food_prices= + =food_quantities= column structure

Same family — country scripts emit multi-column price/quantity
decompositions (market, farmgate, unitvalue_*) that don't map 1-to-1
onto the canonical =Price= / =Quantity=.  Decision needed: which
column is "the" canonical value?

*** =plot_features= — remove or implement?

Same class as =nonfood_acquired= (removed in b75de698).  But
=plot_features= is a more substantial intended table — the full
agricultural plot module.  Removing it closes LSMS-ISA coverage of
ag data; implementing it is a multi-week effort across 13 countries.
Recommend: open a dedicated issue, discuss scope, not autonomously
remove.
