* Lessons learned
- DVC lock contention kills parallel agents sharing one repo; worktrees fix this
- =get_dataframe()= is the only acceptable data access pattern; agents kept reinventing dvc.api.open
- Agents need explicit "do NOT pip install" instruction
- 1h Slurm limit too short for complex features; use 2h
- Dispatch one agent per country (not per feature) to avoid merge conflicts
- Validation gate needs to catch warnings, not just errors
- File GitHub issues for every problem discovered — discovering is half the battle
- Update CONTENTS.org with country-specific quirks as you find them
